# WeFlow -> 夸克/AList -> Hermes 同步交接说明

这份文档用于从当前主机迁移到家里工控机继续部署。当前主机负责从 WeFlow 获取微信聊天记录，并把 ChatLab JSON 上传到夸克；工控机后续负责从夸克拉取文件，并交给 Hermes Agent 每小时整理总结。

## 1. 总体架构

```text
当前主机
  WeFlow HTTP API
  -> weflow-hermes-sync 主机脚本
  -> rclone
  -> 本机 AList WebDAV
  -> 夸克 /WeFlow-Hermes/inbox/

家里工控机
  AList 连接同一个夸克账号
  -> rclone 拉取 /WeFlow-Hermes/inbox/
  -> Hermes Agent 读取本地 ChatLab JSON
  -> 生成摘要
  -> 可选上传到 /WeFlow-Hermes/summaries/
  -> 原始文件移到 /WeFlow-Hermes/processed/
```

AList 的角色是把夸克网盘转换成 WebDAV 文件接口；rclone 是命令行客户端，用来上传、下载、列目录、移动文件；cron 或 systemd timer 负责定时执行脚本。

## 4. 当前主机定时任务建议

当前主机负责每小时把最近消息上传到夸克。建议 cron：

```cron
7 * * * * cd /home/yangsan/weflow-hermes-sync && npm run push >> data/logs/push.log 2>&1
```

添加方式：

```bash
crontab -e
```

说明：

- 第一次运行默认拉最近 1 小时。
- 后续运行读取 `data/state.json` 中的 `lastSyncAt`。
- 脚本会加 120 秒重叠窗口，降低边界消息漏同步风险。
- ChatLab JSON 文件名包含时间窗口和 session id。

## 6. 工控机处理脚本设计

工控机脚本建议命名：

```text
process-weflow-chatlab.sh
```

处理流程：

```text
1. rclone copy quark-alist:/quark/WeFlow-Hermes/inbox/ 到 /srv/hermes/inbox/weflow/
2. 找出本地尚未处理的 *.chatlab.json
3. 逐个交给 Hermes Agent 总结
4. 摘要写入 /srv/hermes/summaries/weflow/
5. 可选上传摘要到 quark-alist:/quark/WeFlow-Hermes/summaries/
6. 处理完成后，把远端原文件移到 processed，或记录本地 processed 清单
```

伪命令：

```bash
rclone copy quark-alist:/quark/WeFlow-Hermes/inbox/ /srv/hermes/inbox/weflow/

for file in /srv/hermes/inbox/weflow/**/*.chatlab.json; do
  # 跳过已经处理过的文件
  # 调用 Hermes
  # 写 summary
done
```

Hermes 调用需要根据工控机上实际命令调整。目标 prompt 可以先用：

```text
你正在读取 WeFlow 导出的 ChatLab JSON。请总结最近一小时微信消息：
1. 重要事件
2. 需要回复的人和内容
3. 待办事项
4. 风险或异常
5. 按时间线整理的简短摘要
```

## 8. 排错记录

### rclone 401 Unauthorized

常见原因：

```text
rclone 保存的是旧 AList 密码
AList 用户没有 WebDAV 权限
AList 连续错误登录触发临时封禁
```

查看 rclone 当前账号：

```bash
rclone config show quark-alist
```

更新 rclone 密码：

```bash
rclone config password quark-alist pass 'AList密码'
```

AList 连续输错后可能返回：

```text
Too many unsuccessful sign-in attempts
```

可以重启 AList：

```bash
sudo systemctl restart alist
```

### AList admin set 后有 token is invalidated

出现：

```text
[del_user_cache_online] error: token is invalidated
```

通常不是密码设置失败，而是清理旧在线 token 时失败。关键看前面是否有：

```text
admin user has been updated
username: admin
password: ...
```

### 夸克/AList 远端 move 不稳定

测试中 `rclone moveto` 出现过：

```text
copy NewObject failed: object not found
```

因此当前主机脚本默认直接上传到 inbox：

```text
WEFLOW_SYNC_UPLOAD_MODE=direct
```

不要优先使用 staging -> moveto，除非后续确认工控机侧 AList/夸克版本支持稳定 move。
